# 進捗管理ルール

読書の進捗を記録・更新するためのルール。

## 進捗ファイル

`progress.md` をプロジェクトルートに配置し、以下のフォーマットで管理する。

```markdown
# 進捗

## 読了・議論済み

- [x] 第◯部 ◯章：タイトル
- [x] 第◯部 ◯章：タイトル

## 未読・未議論

- [ ] 第◯部 ◯章：タイトル
```

## 更新タイミング

以下のタイミングで `progress.md` を自動的に更新する。

- ユーザーが章のまとめを作成した直後 → その章を「読了・議論済み」に移動
- ユーザーが「進捗見せて」「どこまでやった？」と聞いたとき → 現在の `progress.md` を表示
- ユーザーが「◯章読んだ」「◯章終わった」と明示したとき → 該当章を「読了・議論済み」に移動

## 更新時の確認

進捗を更新する際は、ユーザーに確認してから反映する（勝手にチェックを入れない）。

例：
> 14章のまとめができたので、progress.mdの14章を「読了・議論済み」に移してもいいですか？

## 新しい章を始める前に

ユーザーが「◯章お願い」と言ったら、`progress.md` と `summaries/` 配下の関連するまとめを確認してから議論を始める。過去に議論した章と繋がりがあればそれにも触れる。
